Offline AI System for Manufacturing Ops

Factory Maintenance AI System turns repair data into answers that feel fast, calm, and reliable.

This project is designed to support maintenance and technical operations in a 100% offline factory environment. It combines FastAPI, local LLMs, SQL analytics, and vector retrieval so operators can ask real production questions inside a secure LAN without relying on external APIs.

What makes it system-level

Not just a chatbot

This factory maintenance AI system is positioned as a decision assistant. It does not only chat back, but decides when to answer conversationally, when to query repair history, when to retrieve similar repair cases, and when to combine both paths for hybrid reasoning.

Professional angle

Built for constraints that matter

  • Runs inside an air-gapped LAN with zero API cost
  • Supports Thai and English maintenance workflows
  • Uses SQL safety rules before execution
  • Separates structured analytics from knowledge retrieval

Architecture Snapshot

Custom orchestration with multi-pipeline routing

User
  -> FastAPI Orchestrator
     -> Data Readiness / Guardrails / Entity Resolve
     -> Unified Router
        -> CHAT
        -> SQL
        -> VECTOR
        -> HYBRID
        -> META
     -> Response Formatter + Logging
  -> Frontend UI
01

Deterministic + LLM Router

Entity signals and domain rules handle most routing, while the local model resolves only the ambiguous cases.

02

SQL Analytics Path

Repair and PM facts come from generated SQL with safety rules, retry logic, and controlled execution.

03

Vector Knowledge Path

Similar repair cases and knowledge fragments are retrieved through FAISS, embeddings, and reranking.

04

Hybrid Reasoning

Structured analytics identify the top issue first, then retrieval adds supporting evidence and explanation.
