What makes it portfolio-worthy

Not just a chatbot

Elin is positioned as a factory decision assistant. It does not only chat back, but decides when to answer conversationally, when to query repair history, when to retrieve similar repair cases, and when to combine both paths for hybrid reasoning.

Professional angle

Built for constraints that matter

  • Runs inside an air-gapped LAN with zero API cost
  • Supports Thai and English maintenance workflows
  • Uses SQL safety rules before execution
  • Separates structured analytics from knowledge retrieval

Architecture Snapshot

Custom orchestration with multi-pipeline routing

User
  -> FastAPI Orchestrator
     -> Data Readiness / Guardrails / Entity Resolve
     -> Unified Router
        -> CHAT
        -> SQL
        -> VECTOR
        -> HYBRID
        -> META
     -> Response Formatter + Logging
  -> Frontend UI
01

Deterministic + LLM Router

เริ่มจาก entity signal และ domain rules ก่อน แล้วให้ LLM ช่วยเฉพาะส่วนที่กำกวมมากขึ้น

02

SQL Analytics Path

ดึงข้อเท็จจริงจาก repair/PM database ด้วย generated SQL ที่มี guardrails และ retry safety

03

Vector Knowledge Path

ค้นหา repair logs และ knowledge fragments ด้วย FAISS + embeddings + reranking

04

Hybrid Reasoning

ใช้ structured data หา top issue ก่อน แล้วเอาไปหาหลักฐานเชิงข้อความเพื่ออธิบายสาเหตุและแนวทางแก้

How to talk about Elin in interviews or GitHub

01

The problem

โรงงานมีข้อมูลซ่อมเยอะ แต่ทีมใช้งานต้องการคำตอบที่สรุปไว ตรง และอธิบายเชิงสาเหตุได้

The approach
